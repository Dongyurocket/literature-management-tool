# CHANGELOG

本文件记录项目已发布版本的重要更新，按版本倒序排列。

更早的中文发布说明可参考 `docs/releases/` 目录。

## v1.0.0 - OCR 移除与版本稳定化

### 本版重点
- 完整移除 OCR / Umi-OCR 相关能力，包括设置项、工具箱入口、后台服务与 PDF OCR 回退流程
- 保留 PDF 文本提取、全文检索、批量导出、重复检测、统计报表与维护工具等核心能力
- 清理 README 与当前版本说明，避免界面和文档继续出现 OCR 相关信息

### 兼容性说明
- 旧版 `settings.json` 中遗留的 OCR 字段会在读取时自动忽略，不影响现有文库继续使用
- 本次发布不涉及数据库结构升级，已有文献库与附件目录可直接沿用

## v0.9.0 - 附件修复、短标题与文库自定义路径

### Bug 修复
- 修复附件添加因 Windows 路径过长（MAX_PATH 260 字符限制）导致静默失败：附件现在统一存放在文库根目录，不再按文献创建子文件夹
- 修复附件添加全部失败时仍显示"成功"：根据实际结果区分 success / warning 提示，全部失败时显示具体错误原因
- 附件处理循环异常捕获从 `OSError` 扩展为 `Exception`，避免非文件系统异常中断整个批量操作

### 新功能
- 新增"短标题"字段：填写后 PDF 批量重命名将优先使用短标题生成文件名，避免长标题导致路径超限
- 新增"译题"可选列：文献列表支持显示译题列，需在列设置中手动启用
- 文库管理支持自定义文库目录：新建文库时可选择自定义路径（留空则使用默认路径）
- 文献列表新增"短标题"可选列

## v0.8.0 - 元数据合并修复与文库管理增强

### Bug 修复
- 修复抓取元数据后合并功能：确认合并后获取到的元数据无法正确写入文献记录，原因是 `_refresh_table` 中的 flush 用旧表单值覆盖了刚合并的数据；现在在读取详情和应用合并前各增加一次 flush
- 修复附件"复制到库"后静默失败：`_add_attachments` 仅捕获 `ValueError`，未覆盖文件操作可能抛出的 `OSError` / `PermissionError` / `FileNotFoundError`；现在改为 `except Exception` 并对单个文件失败做跳过处理
- `compute_checksum` 在文件不存在时返回空字符串，避免异常

### 文库管理增强
- 新增"删除文库"功能，支持仅删除注册记录或同时删除文库目录和数据库文件
- 文库管理对话框新增"文献数"列，直观展示各文库文献数量
- 隐藏"数据库"列（用户不需要看数据库路径），调整列宽
- 当前活动文库行蓝色高亮，已归档行灰色
- 双击文库行即可快速切换
- 切换文库前自动保存未保存的编辑

### 备份恢复改进
- 恢复备份前自动创建 `auto_backup_before_restore.zip`，防止误操作导致数据丢失

## v0.7.2 - 打开所在文件夹与工具箱 UX 优化

- 附件区"定位文件"改为"打开所在文件夹"，直接打开文件所在目录，不再依赖 `explorer /select,`
- 未选择文献时的 toast 提示更明确："请先在文献列表中选择一条或多条记录。"
- 全文检索：空查询拦截 + 零结果 / 有结果状态反馈
- 复制 GB/T：生成失败时提示检查标题、作者等必填字段
- PDF 重命名预览：无需修改项标注"文件名已符合规则"
- 维护工具-路径修复：修复完成后区分"全部恢复"与"仍有未解决项"
- 重复检测：零结果 toast 停留时长加长
- 导入中心：扫描前显示"正在扫描，请稍候..."，扫描后区分有结果 / 无结果

## v0.7.1 - 资源管理器定位修复与附件交互优化

- 修复"定位文件"打开错误位置：`reveal_path` 改用字符串格式调用 `explorer /select,`，解决路径含空格时 Python `list2cmdline` 引号包裹导致 Explorer 无法识别 `/select,` 标志的问题
- 修复 `resolve_path` 路径对称性：拼接时统一使用 `root.resolve()`，与 `_store_path` 基准一致
- 未选中附件时点击"打开"或"定位文件"改为 toast 提示，不再静默失败
- 附件选中状态与按钮启禁联动，取消选中后禁用操作按钮
- 新增路径含空格、中文字符的定位测试用例

## v0.7.0 - 附件定位修复与交互增强

- Delete 键快捷删除附件（作用域限定在附件列表）
- 删除附件后自动选中相邻项
