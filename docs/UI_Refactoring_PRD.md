# 📚 文献管理软件 UI重构 PRD 与技术方案

这份针对《文献管理软件（Literature Management Tool）》的**UI重构PRD（产品需求文档）与技术方案**，旨在将其从传统的、基础的 `tkinter` 界面，升级为具备**现代化、高颜值、沉浸式体验**的专业级桌面应用。

重构设计的核心原则是：**信息层级清晰（Clarity）、交互符合直觉（Intuitive）、视觉现代化（Modern Aesthetics），以及底层架构的高可维护性（Maintainability）**。

---

## 第一部分：现状分析与重构目标
**当前痛点（基于 `tkinter` 原生实现）：**
1. **视觉过时**：原生 Tkinter 组件带有强烈的年代感（Win95/Win7 风格），缺乏现代 OS 的阴影、圆角、平滑动画和毛玻璃效果。
2. **交互生硬**：原生弹窗（如 `messagebox`）会阻塞用户操作，缺乏全局 Toast 提示、平滑过渡等优雅交互。
3. **高分屏（High-DPI）支持弱**：在 4K 显示器下，Tkinter 容易出现字体发虚或组件比例失调。
4. **代码耦合度高**：当前的 `ui.py` 高达 1700+ 行，视图（View）与事件处理逻辑深度绑定，难以进行自动化测试和后续扩展。

**重构目标：**
打造一个对标（甚至超越）Zotero 6 / Mendeley 视觉体验的**下一代本地文献管理工具**，提供极简、专注的沉浸式科研环境。

---

## 第二部分：UI/UX 产品需求文档 (PRD)

### 1. 设计语言与风格 (Design Language)
*   **整体风格**：采用 **Windows 11 Fluent Design** 或 **扁平化毛玻璃风格 (Glassmorphism)**。以无边框窗口、大圆角、微渐变和软阴影为主。
*   **主题系统 (Theme)**：全面支持 **深色模式 (Dark Mode)** 和 **浅色模式 (Light Mode)**，并支持跟随系统自动切换。夜间阅读和管理文献时有效保护视力。
*   **排版与色彩**：
    *   *主色调*：科研蓝（#005FB8）或 典雅青，传递专业、宁静的视觉感受。
    *   *字体*：采用系统原生无衬线字体（Windows 默认 Segoe UI / 微软雅黑），文献列表支持动态缩放。

### 2. 核心布局规划 (三栏式经典布局)
界面划分为顶部工具栏和下方经典的“左-中-右”三栏弹性布局（支持拖拽调整列宽）。

#### A. 顶部全局导航区 (Header/TitleBar)
*   **沉浸式标题栏**：将系统标题栏与工具栏融合（类似浏览器），节省纵向空间。
*   **全局搜索框 (Ctrl+F)**：居中放置，圆角搜索框。支持全局模糊搜索（标题、作者、年份、标签、笔记全文）。
*   **快捷操作区**：右上角放置“添加文献(下拉)”、“同步/导入”、“设置”图标按钮。

#### B. 左侧边栏：导航与分类 (Left Panel)
*   **视觉**：半透明毛玻璃背景或浅灰色底，区分主次层级。
*   **内容**：
    *   *系统分类*：所有文献、最近添加、正在阅读、我的收藏、回收站（配以精细的线性图标）。
    *   *用户文集/标签树*：支持无限层级的树状文件夹（Tree结构），支持拖拽文献到特定文件夹。

#### C. 中间区：文献数据列表 (Center Data Grid)
*   **视觉**：极简的数据表格，隐藏繁杂的网格线，采用斑马纹或 Hover 悬浮高亮。
*   **特性**：
    *   **状态指示器**：阅读状态（未开始、在读、已读）用不同颜色的状态小圆点（红/黄/绿）表示。
    *   **自定义列**：标题、作者、年份、期刊、DOI等。支持表头拖拽排序、右键隐藏/显示列。
    *   **富文本高亮**：搜索时，匹配到的关键字在列表中直接高亮（黄色背景）。

#### D. 右侧边栏：属性面板与沉浸阅读区 (Right Inspector)
*   **Tab 页签切换**：分为 [信息 (Metadata)]、[笔记 (Notes)]、[附件 (Attachments)]。
*   **信息 Tab**：优雅的表单设计，支持**就地编辑（Inline Editing）**，点击标题即可直接修改，失去焦点自动保存，无需专门的“编辑/保存”按钮。
*   **笔记 Tab**：集成 **Markdown 所见即所得 (WYSIWYG)** 编辑器，支持代码块、加粗、高亮。
*   **附件 Tab**：显示 PDF 缩略图和文件大小，支持一键调用外部阅读器或内部预览。

### 3. 人性化交互细节 (Micro-interactions)
*   **拖拽导入**：直接将 PDF 拖入主界面，全屏出现“松手即可导入并解析”的毛玻璃遮罩覆盖层。
*   **非阻塞提示 (Toast)**：导入成功、DOI 获取失败等提示不再弹确认框，而是从右下角滑出优雅的 Toast 通知，3秒后自动消失。
*   **骨架屏加载 (Skeleton Screen)**：进行大量数据库检索或导入扫描时，列表显示呼吸波动的骨架屏，而非整个软件卡死。

## 第三部分：UI 重构技术方案

鉴于当前项目为 Python 生态，且底层逻辑（`db.py`, `metadata_service.py` 等）已封装良好，推荐以下三种技术路径。

### 核心推荐方案：PySide6 (Qt) + PyQt-Fluent-Widgets + MVVM 架构

这是目前 Python 桌面端能达到**商业级颜值与性能**的最优解。

#### 1. 为什么选择这套技术栈？（合理性解释）
*   **极高的颜值**：`PyQt-Fluent-Widgets` 提供了一整套开箱即用的 Windows 11 风格组件（圆角、亚克力效果、现代化输入框、Toast 提示），极大降低了自己写 QSS（Qt样式表）的成本。
*   **性能优异**：Qt 的 `QTableView` 采用真正的 Model-View 架构。面对一万条文献数据，Tkinter 的 Treeview 会卡顿，而 Qt 的 `QAbstractTableModel` 只渲染可视区域，滑动如丝般顺滑。
*   **与现有系统兼容**：完全支持调用你现有的 Python 爬虫/解析/数据库逻辑。
*   **高分屏原生支持**：只需一行代码 `QApplication.setHighDpiScaleFactorRoundingPolicy` 即可完美适配 4K 屏幕。

#### 2. 架构分层重构 (MVVM 设计模式)
废弃现有的单文件 `ui.py` (1700+ 行)，进行解耦重构：

*   **Model（数据模型层）**：复用现有的 `db.py` 和业务 Services，保持不动。
*   **ViewModel（视图模型层）**：
    *   新增 `viewmodels/` 目录。
    *   作用：负责将 Model 的数据转换为 UI 能直接绑定的状态，如将数据库的 `reading_status` 转换为颜色代码。处理耗时操作的异步线程（防止界面卡死）。
*   **View（视图层）**：
    *   新增 `views/` 目录。只负责界面的排版、动画和信号（Signal）绑定。
    *   `main_window.py`: 主窗口框架。
    *   `components/`: 抽离复用组件（如 `search_bar.py`, `pdf_previewer.py`）。

#### 3. 关键技术难点与对策
*   **长耗时操作导致界面假死**：
    *   *现状*：现有的 `scan_import_sources` 或 `lookup_doi` 若同步执行会阻塞 Tkinter 主循环。
    *   *对策*：在 Qt 中使用 `QThread` + `Signals/Slots`，或使用 `QThreadPool` 将网络请求和 PDF 扫描放入后台，前台通过信号更新进度条（ProgressRing）。
*   **安装包体积问题**：
    *   *现状*：引入 PySide6 会导致 PyInstaller 打包体积增加约 40-60MB。
    *   *对策*：通过优化 `.spec` 文件，排除无用的 Qt 模块（如 QtWebEngine, Qt3D 等），配合目前的 `build_windows.ps1` 和 Inno Setup（`.iss`）进行高压缩率打包。

---

### 备选方案 B：低成本平替（不改变框架）— CustomTkinter / ttkbootstrap
如果团队希望**最小化重构工作量**，不愿学习 Qt 框架。

*   **方案**：使用 `CustomTkinter` 或 `ttkbootstrap` 替换原生的 `tkinter` 组件。
*   **优点**：几乎不需要修改底层事件绑定，只需将 `tk.Button` 换成 `ctk.CTkButton`，即可获得暗黑模式和圆角外观。
*   **缺点**：依然受限于 Tkinter 的性能瓶颈（如长列表滚动性能），缺乏真正的高级动画机制，右键菜单和复杂表格定制依然困难。

## 第四部分：重构实施路线图 (Roadmap)

建议分四个 Sprint（迭代阶段）进行平滑重构，避免“大爆炸”式的重写导致功能倒退。

**Phase 1: 架构解耦与准备 (1周)**
*   在不改变现有 Tkinter 界面的前提下，将 `ui.py` 中混杂的数据库操作抽取到 `literature_manager/controllers` 中。
*   搭建 PySide6 基础工程模板，配置 PyInstaller 以支持多框架并行测试。

**Phase 2: 主框架与导航构建 (1周)**
*   使用 PySide6 + PyQt-Fluent-Widgets 搭建无边框主窗口。
*   实现深浅主题切换、左侧边栏导航树（对接数据库读取分类）。

**Phase 3: 核心数据区与详情重构 (2周)**
*   实现中间的文献 DataGrid（继承 `QAbstractTableModel`，实现分页/按需加载，提升性能）。
*   实现右侧详情面板，对接现有的元数据修改和 Markdown 笔记加载逻辑。
*   实现通过拖拽文件导入文献的拖放事件（`dragEnterEvent`, `dropEvent`）。

**Phase 4: 细节打磨与测试发布 (1周)**
*   替换原有的弹窗为 Toast 和毛玻璃 Dialog。
*   补充异步线程池，优化导入、排重（`dedupe_service.py`）时的进度提示。
*   更新 `build_windows.ps1` 和 `LiteratureManagementTool.iss`，测试 Windows 环境下的一键安装与运行。

---

### 结语
这套《UI重构PRD+技术方案》不仅聚焦于把软件“变好看”，更致力于将项目推向**工程化、模块化**。采用 **PySide6 + MVVM** 的方案，将使得软件具备极高的可扩展性，未来甚至可以轻松加入“内置 PDF 标注”、“知识图谱可视化”等高级特性，为这款文献管理工具注入长久的生命力。
