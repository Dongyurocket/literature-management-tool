## Literature management tool v0.3.1

这一版聚焦“中文文献元数据补全 + Qt 交互稳定性”两条主线。我们补上了高校图书馆 OpenURL 与 CNKI 的在线元数据解析能力，同时修复了多个异步任务完成后 UI 仍显示“进行中”的问题。

### 下载与校验

- Release 页面：[v0.3.1](https://github.com/Dongyurocket/literature-management-tool/releases/tag/v0.3.1)
- 安装包：`Literature-management-tool-v0.3.1-Setup.exe`
- SHA256：`0647C91D3182BAC601CEC779747C971B8D7A03B3BEDEB18CAFA055B3BAAA2BCE`

### 本版重点改进

- 元数据回退链路扩展为 `Crossref / DataCite / OpenAlex / CNKI / USTC OpenURL / THU OpenURL / OpenLibrary / Google Books`
- 支持从 OpenURL 解析器与 CNKI 抓取“可用字段”，未命中字段允许为空并继续人工编辑
- 优化元数据抓取体验：支持手动输入 DOI / ISBN，并允许手动标识优先于已有字段
- 修复 Qt 异步任务状态残留问题（如 PDF 重命名、Umi 安装下载、软件更新下载）
- 增加 Qt 交互回归测试，覆盖嵌套任务、回调异常与收尾异常等高风险路径
- Umi-OCR 配置与端口读取路径已验证，可识别已安装的 `Umi-OCR.exe`

### UI 变化

- 新增 v0.3.1 截图：主界面、设置页、更新对话框（见 `docs/screenshots/`）
- 设置页增加新的元数据源选项：`知网`、`中科大图书馆 OpenURL`、`清华图书馆 OpenURL`

### 打包与验证

- 单元测试：`python -m unittest discover -s tests -v`（通过）
- 打包命令：`powershell -ExecutionPolicy Bypass -File .\scripts\build_windows.ps1 -Version 0.3.1`（通过）
- 安装包烟雾验证：已完成静默安装、启动存活检查、静默卸载（通过）

### 升级说明

1. 下载并运行 `Literature-management-tool-v0.3.1-Setup.exe`
2. 按安装向导完成升级
3. 原有文献数据库与文献库目录默认保留，不会被卸载程序主动删除

### 已知说明

- `Umi-OCR` 首次下载安装体积较大，建议保持网络稳定
- PDF 元数据与全文提取仍为尽力而为能力，建议配合人工校对
