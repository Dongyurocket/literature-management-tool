## Literature management tool v1.0.0

这一版标志着项目进入一个更聚焦、更稳定的阶段：我们移除了 OCR / Umi-OCR 相关能力，把产品能力集中到本地文献管理、元数据补全、全文检索、批量导出、查重合并与维护工具上。

### Release 页面

- Release 页面：[v1.0.0](https://github.com/Dongyurocket/literature-management-tool/releases/tag/v1.0.0)
- 安装包：`Literature-management-tool-v1.0.0-Setup.exe`

### 本版重点

- 删除 OCR / Umi-OCR 设置项、安装入口、工具箱按钮与后台服务模块
- PDF 处理链路回归为原生文本提取，减少维护复杂度与外部依赖
- 保留并延续文库管理、元数据抓取、全文检索、模板导出、统计报表、重复检测、备份恢复等核心能力
- 更新 README、CHANGELOG 与版本号，统一当前产品说明与发布流程

### 兼容性说明

- 旧版配置文件里的 OCR 字段会在读取时自动忽略，不需要手动清理
- 已有文献库、数据库和附件目录可直接沿用
- 本次发布不包含数据库 schema 变更

### 验证情况

- `python -m pytest -q`
- `python -m compileall literature_manager tests`

### 已知说明

- PDF 元数据抽取与文本提取仍属于尽力而为能力，部分文档仍需人工补充信息
- 查重合并建议在预览后再执行
