## Literature management tool v1.1.0

这一版把“跨设备同步”从手工折腾升级为官方支持能力：现在可以直接把整个工作区放进百度网盘等同步目录，在多台设备之间同步文库数据库、设置、附件和索引定位信息。

### Release 页面

- Release 页面：[v1.1.0](https://github.com/Dongyurocket/literature-management-tool/releases/tag/v1.1.0)
- 安装包：`Literature-management-tool-v1.1.0-Setup.exe`

### 本版重点

- 新增“跨设备同步友好模式”，可在设置中切换到百度网盘、OneDrive、Dropbox、Syncthing 等同步工作区
- 工作区内统一同步 `library_registry.json`、各文库的 `settings.json`、`library.sqlite3` 与 `library_files`
- 文献库目录优先按 profile 相对路径或 workspace 相对路径保存，减少不同设备盘符差异导致的路径失效
- 同步模式下自动禁用 `link`，默认改为 `copy`，避免只记录本机绝对路径
- SQLite 增加 `busy_timeout`，关闭前执行 checkpoint，提升同步盘场景下的稳定性
- README、CHANGELOG 与测试覆盖同步更新，明确官方推荐的跨设备用法

### 使用建议

- 推荐同步整个工作区目录，不要只同步 `library_files`
- 第二台设备首次使用时，在设置中指向同一个同步工作区即可
- 建议同一时刻只在一台设备上编辑，等待同步完成后再切换到另一台设备

### 验证情况

- `python -m pytest -q` -> `71 passed`
- `python -m compileall main.py literature_manager tests`

### 已知说明

- 当前支持的是“同步工作区”模式，不包含实时协同、冲突合并或多端同时编辑保护
- 如果目标目录非空但不是有效工作区，程序会拒绝直接迁入，避免误覆盖已有文件
